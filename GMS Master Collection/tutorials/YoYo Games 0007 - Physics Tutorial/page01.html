Physics in GameMaker

The aim of this tutorial is to introduce the experienced user to how the physics system in GameMaker:Studio is set up. As such, it does not cover all aspects of what is possible using physics, but rather introduces you to some of the core concepts so that you can then experiment and create things afterwards with a firm understanding of the basics.

The game we are going to make, is a physics based version of "Asteroids", with a player ship, bullets, a bomb and (of course) asteroid objects, and each will have it's own physics which will help to create a fun and challenging game!

Before we get started, it is worth noting that the physics system in GameMaker:Studio is completely separate from the tradition movement and collision system that you may be used to. The moment that you tell GameMaker:Studio that you will be using physics in your rooms, all interactions between objects will be done through the physics system and previously available "built in" variables like speed and image_angle are no longer used as they were before, instead there are a whole new set of physical world variables and constants that we will need to use.

The game will use some special resources created for this purpose. To find them on your computer, please go to the top of the GameMaker screen and select the Help drop-down menu. You will see an option marked Open Project in explorer. If you select that then the standard file explorer will open showing the contents of you GameMaker project directory, all resources are stored here.
