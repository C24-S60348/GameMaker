Summary

That brings us to the end of this tutorial. Hopefully you will have learned enough to create your own saving mechanisms without too much fuss! However, its worth just revising some of the important points that this tutorial has brought up:

The File System

  1. GameMaker:Studio has TWO areas for loading from: the bundle and local storage
  2. You can only save to the local storage
  3. If you change an included file, it is copied to local storage and the copy is what is changed
  4. Your game is sandboxed and so you cannot access files outside of its area

Ini Files

  1. For small amounts of data use ini files
  2. Have a "base" file included with your game to make reseting values easier
  3. Ini files store information as key/value pairs
  4. If a section, or key, does not exists, the ini functions will return a "default" value
  5. You can only have one ini file open at a time (although you can use multiple ini files with different names)

Text Files

  1. Use text files for large amounts of data or data that does not fall into a key/value pair
  2. You can open a text file for reading, writing or appending
  3. Writing to a text file wipes it of all data first
  4. Appending a text file will add data to whatever has previously been saved
  5. There are special functions to check when you have reached the end of a line or the end of the file
  6. You can have multiple text files open at any one time
This tutorial has only covered the basics of saving and loading data with GameMaker:Studio but it should be enough to get you started with your own. We suggest that you also revise the manual that comes with your copy of the program as it contains a complete list of all available file functions as well as some small practical examples of how each function is used.
