Ini Files

Of all the file types that can be created, saved and loaded by GameMaker:Studio, arguably the easiest to use are *.ini files. These files are commonly used by all windows programs to store small "packets" of data, especially configurations and user options. Internally, they have a simple structure, where the information they store is split into sections, where each section can hold a number of key/value pairs. So, for example, an ini file for general game options may have this structure:
[Options]
music = 1
sound = 1
diff = 2
name = Anon
As you can see, the section is called "Options" and in that there are various keys (music, sound etc...), each with its own value, which can be either a real or a string data type.

Now, in our example "game", you will see that we have included the ini file that we are going to use in the "Included Files" assets of the resource tree. GameMaker:Studio does not actually require you to include the files that you wish to write to, as when you try and write information to an ini that does not exists it is created for you. But, and this is important, by including a base ini file with your game you can setup easily "reset" mechanisms (the next part of this tutorial explains this), and also guarantee that the file created is the correct format, allowing users to save information from different localisations.

To create an ini file for any game, it is a simple case of opening up the Windows Notepad, setting up the base sections with their key/value pairs and then saving the file as an *.ini (as opposed to a *.txt) file. Note that when saving the file you should use "save as" and from the save dialogue that prompts you for the name to use you should make sure that the Encoding selected is UTF8 otherwise you may get errors when dealing with saved data from people in localisations other than US/UK.

The "base" ini file that is included with the tutorial has this format:
[Sound]
music = 1
sound = 1

[Controls]
0 = 38
1 = 40
2 = 37
3 = 39
4 = 32
Remember, when we start to save information, this ini will be copied over to the local storage area for the device that we are running on, and the base file will remain un-modified in the game "bundle". Continue on to the next page now to program some simple state saves.
