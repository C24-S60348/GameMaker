Isometric And GameMaker

Now that we have covered the ideas behind displaying an isometric grid we'll move onto how to construct a simple isometric game using GameMaker. For this basic form of an isometric game we will allow the user to manipulate the visibility of the grid and alter the tiles on the grid using a simple HUD that users can drag and drop tiles from onto the grid.

For this tutorial we wish to allow the user the facility to manipulate the portion of the grid visible to them and thus we want to allow them to pan the grid across the screen and zoom in/out of the grid.

To do this we will add a room that is sized at 1280x960 and setup a view onto the room that is 640x480, giving the view an initial (x, y) of (320, 240). By manipulating the settings of the view - xview, yview, wview and hview - we can allow panning and zooming.

The objects we're going to use to build our game are as follows:

  • objIsoGrid - The object responsible for maintaining and drawing the isometric grid.

  • objGridDimensions - The object encapsulating the data for the width and height of the grid in elements.

  • objMouseControl - The object that interprets mouse input and manipulates the view setting.

  • objHUD - The object that controls the layout of the HUD

  • objHudTile - This is a base class object that specific HUD elements inherit from that defines common behaviour for their operation.
The objects added to the room (in any arbitrary location) will be objIsoGrid, objMouseControl and objHUD.

For this tutorial we are also going to create two sprites: sprRoad and sprGrass using Iso_Road.png and Iso_Grass.png. We will associate these sprites with two objects objRoadTile and objGrassTile that will use the objHudTile object as their parent. To setup a parent relationship with another object, select that object and alter the "Parent: " object option.
