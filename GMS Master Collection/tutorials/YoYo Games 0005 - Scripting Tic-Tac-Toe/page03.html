Create the Objects

We will now setup the room and the objects we need to control the game.

To setup the room, select Resources and Create Room. We will be using a width of 640 and height of 480. Select the backgrounds option in the Room Properties window and select the background image we created earlier.

The first object we will create is an object whose responsibility it is to represent the grid the game is played on and handle the player's interactions with the grid. Select Resources and Create Object. Rename the new object to objGameBoard and select it.

Next, add a Create event for the object and add an Action that performs the Execute a piece of code action. Edit the code and simply add the following piece of script: scrGameInit(). We will implement the script this code calls later on.

For the purposes of this tutorial we will keep the code that is executed by the actions added to each object down to the bare minimum and will simply call through to the relevant script that is added later on. This maintains a certain level of cleanliness to the project by ensuring that all script code is found in a simple, consistent location.

The other events we need to add for this object are:

Event Script Called
Step scrGameUpdate()
Global Mouse Left Pressed scrMousePressed()
Draw scrGameBoardDraw()
Before proceeding further, return to the room we previously created and add an instance of this object to the room - any location in the room will do. This is the only object we need to add to the room and thus concludes the setup necessary for the room.

We need to create a further three objects:
  • objBoardCell : represents the contents of a cell on the board
  • objBoardLayout : represents the location in the room of the board and the size of the grid cells
  • objGameDifficulty : represents the object used to select the difficulty setting for the AI
Note that the object for the board layout, objBoardLayout, may seem extraneous, but it is used to separate out how the display of the grid is presented from the logic of the board thus providing a single, clean location where we can modify the layout of the game according to our assets. It is notionally present as an introduction to the idea of the single responsibility principle of software engineering and an introduction as to how we can use an object to essentially encapsulate what is otherwise global data.

Note also that the object objGameDifficulty has the sprite sprDifficulty attached to it although its rendering we will handle via script for the sake of consistency.

The event/script pairs for the additional 3 objects are as follows:

objBoardCell:
Event Script Called
Create scrBoardCellInit()
Draw scrBoardCellDraw()
objBoardLayout:
Event Script Called
Create scrBoardLayoutInit()
objGameDifficulty:
Event Script Called
Create scrGameDifficultyInit()
Draw scrDifficultyDraw()
Left Pressed scrDifficultySelected()
